# Google Algorithm Update History

Key updates that shaped current SEO. Understanding these helps diagnose why rankings changed.

---

## Active Core Ranking Systems (as of 2025)

These are always running — not one-time updates:

| System | Introduced | What It Targets |
|--------|-----------|----------------|
| PageRank | 1998 (ongoing) | Link authority; how pages link to each other |
| Panda (now core) | 2011 → 2015 | Thin, low-quality, duplicate content |
| Penguin (now core) | 2012 → 2016 | Link spam, manipulative link building |
| Hummingbird | 2013 | Semantic search; query meaning vs. keywords |
| RankBrain | 2015 | AI-based query/concept matching |
| Mobile-first indexing | 2016–2023 | Mobile version used for ranking (fully rolled out) |
| BERT | 2019 | Natural language understanding |
| Passage Ranking | 2021 | Individual page sections can rank |
| MUM | 2021 | Multitask language understanding (specific features) |
| Helpful Content System | 2022 → 2024 | People-first content; now part of core |

## Types of Announced Updates

### Core Updates
Happen several times per year. Broadly reassess content quality and E-E-A-T signals across the web. Both winners and losers emerge.

**If you're hit by a core update:**
- No single "fix" — it's a holistic quality assessment
- Improve E-E-A-T, content depth, page experience, and helpful content signals
- Recovery typically requires waiting for the next core update
- Check the ranking updates log: status.search.google.com

### Spam Updates
Target specific spam tactics. More targeted than core updates.

**Common spam tactics they target:**
- Scaled content abuse (mass AI-generated thin content)
- Link spam (paid links, link networks)
- Site reputation abuse (third-party spam content on authoritative sites)
- Cloaking (showing different content to Google vs. users)

### Reviews Updates (Merged into Core 2023)
Originally targeted review-style content. Rewarded content showing:
- First-hand experience with the subject
- Specifics (measurements, testing methodology, actual usage)
- Unique insights vs. what manufacturers/others say
- Multiple comparison subjects

## How to Check if an Update Affected You

1. Note the date when your traffic dropped in Search Console
2. Go to: status.search.google.com/products/rGHU1u87FJnkP6W2GwMi/history
3. Match dates
4. Read Google's description of the update
5. Assess: does the description match your site's characteristics?

**Update coincidence vs. causation:**
Traffic drops not matching an update date usually indicate technical issues, not algorithm changes. Check robots.txt, noindex tags, and manual actions first.

---

## Recovery Principles After a Core Update

1. **Don't panic-publish or panic-delete** — Bulk content changes don't reset the algorithm's view of your site
2. **Improve content quality on the pages that dropped** — more depth, more E-E-A-T signals, better answers
3. **Add clear author information** — bylines, bios, credentials
4. **Fix thin pages** — either improve them substantially or use noindex
5. **Improve page experience** — Core Web Vitals, mobile usability
6. **Wait for the next update** — recovery is confirmed when the next core update rolls out

Recovery is measured in months, not weeks.
